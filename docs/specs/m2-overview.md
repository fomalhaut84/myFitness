# 마일스톤 2: 상세 페이지 + 데이터 확장

## 개요

마일스톤 1에서 기본 대시보드와 데이터 싱크를 완성했다.
마일스톤 2에서는 활동/수면 상세 페이지를 강화하고, 미활용 Garmin 데이터를 추출하여 노출한다.

## 이슈 목록

### M2-1: DB 스키마 확장 + 싱크 fetcher 보강
- **목적**: rawData에만 저장되던 Garmin 데이터를 DB 컬럼으로 추출
- **Activity 추가**: 케이던스, 보폭, 수직진동, 지면접촉시간, 유산소/무산소 TE, 호흡수, 랩수, 스플릿
- **SleepRecord 추가**: SpO2, 호흡수(평균/최저/최고), 수면 스트레스, 배터리 변화, 안정시 심박, 야간 HRV, 점수 세부
- **DailySummary 추가**: SpO2(평균/최저), 호흡수, 스트레스 세부 시간(고/중/저), 배터리 충전/소모
- **상태**: PR #36 (진행 중)

### M2-2: 활동 상세 페이지 강화
- **목적**: 러닝 데이터를 심층 시각화
- **킬로미터 스플릿**: 각 km별 페이스/HR/고도 테이블 + 페이스 바 차트
- **러닝 다이나믹스**: 케이던스, 보폭, 수직 진동, 지면접촉시간
- **추가 지표**: 유산소/무산소 TE, 호흡수
- **AI 평가**: 해당 활동 데이터 기반 AI 한줄 평가 버튼

### M2-3: 수면 상세 페이지 신규
- **경로**: `/sleep/[date]`
- **수면 요약**: 총 시간, 점수, 시작/종료, 배터리 변화량
- **수면 단계 바**: 깊은/얕은/REM/깨어남 비율
- **수면 점수 세부**: 총시간, 스트레스, 깨어남횟수, REM%, 깊은수면%, 얕은수면%, 뒤척임
- **바이탈 지표**: SpO2, 호흡수(평균/최저/최고), 수면 스트레스, 안정시 심박, HRV
- **AI 평가**: 해당 수면 데이터 기반 AI 한줄 평가

### M2-4: 기존 UI에 신규 지표 노출
- **대시보드**: SpO2 요약 카드 추가
- **일일 통계**: 스트레스 세부 분포 (고/중/저 스택), SpO2 30일 추세
- **심박 페이지**: 호흡수 30일 추세 추가

## 기술 결정

- 스플릿 데이터는 `splitSummaries Json?` 필드에 배열로 저장
- 수면 점수 세부는 `sleepScoreDetails Json?` 필드에 객체로 저장
- Garmin 스트레스 시간은 초 단위로 반환 → 분 단위로 변환 후 저장
- 수면 상세 페이지는 날짜(YYYY-MM-DD)를 URL 파라미터로 사용

### M2-5: 시간대별 AI 리포트 + 지표 평가 보완
- **목적**: 시간에 따라 변하는 지표(바디배터리 등) 평가 일관성 확보
- **모닝 리포트** (08:00 KST): 수면 분석, 회복 상태, 운동 추천
- **이브닝 리포트** (23:00 KST): 하루 활동 정리, 회복 필요성, 내일 계획
- **시스템 프롬프트**: 시간대별 지표 해석 가이드 (바디배터리 충전/소모 기준 등)
- **UI**: 대시보드 상단 리포트 요약 + /reports 이력 페이지
- **DB**: AIAdvice.reportDate 추가, category에 morning_report/evening_report
- **MCP 도구**: _context 필드로 시간대 바이어스 방지
- **상세 스펙**: `docs/specs/m2-daily-reports.md`

### M2-6: 텔레그램 봇
- **목적**: myFinance 패턴으로 텔레그램 봇 구현
- **커맨드**: /today, /run, /sleep, /weight, /sync, /report, /ai, /reset
- **자연어**: 식단 입력 감지, AI 질문 fallback
- **알림**: 모닝/이브닝/주간 리포트 자동 전송
- **배포**: grammY + 별도 PM2 프로세스 + esbuild 번들
- **상세 스펙**: `docs/specs/m2-telegram-bot.md`
