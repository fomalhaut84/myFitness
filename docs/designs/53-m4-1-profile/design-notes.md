# [M4-1] 프로필 편집 페이지 디자인 노트

## 범위

단일 사용자용 설정 페이지. 신규 경로 `/settings/profile`.
복잡한 프로토타입 불필요 (간단한 폼).

## 레이아웃

데스크톱/모바일 공통, 중앙정렬 max-width ~640px.

```
┌─────────────────────────────────┐
│  ← 대시보드                      │
│                                 │
│  프로필 설정                     │
│  ─────────────────────────      │
│                                 │
│  [기본 정보]                     │
│   이름           [________]     │
│   생년월일       [YYYY-MM-DD]   │
│   키 (cm)        [____]         │
│   목표 체중 (kg) [____]         │
│                                 │
│  [심박 Zone] ⓘ 도움말           │
│   최대 심박 (bpm)    [176]      │
│   LTHR (bpm)         [157]      │
│   LTHR 페이스 (분/km) [5:21]    │
│                                 │
│  [칼로리 목표]                   │
│   일일 칼로리 목표   [1890]     │
│                                 │
│  [저장] [취소]                   │
└─────────────────────────────────┘
```

## 상호작용

- 저장 버튼: PATCH `/api/profile`, 성공 시 토스트 + 대시보드 이동 링크
- 입력 검증 (Zod):
  - maxHR: 100~220
  - lthr: 80~maxHR
  - lthrPace: "M:SS" 형식 (분:초)
  - targetCalories: 500~5000
- 저장 중에는 버튼 비활성 + 스피너
- 필드 옆 도움말 아이콘 (ⓘ) → tooltip:
  - maxHR: "측정 방법: 최대 노력 런 마지막 1분 평균, 또는 Garmin 최고 기록"
  - LTHR: "20분 TT 평균 심박 × 0.95, 또는 Garmin 트레이닝 스테이터스"

## 진입점

- 사이드바 "설정" 신규 메뉴 (기어 아이콘)
- 또는 대시보드 상단 사용자 메뉴 (향후 확장 가능)

## 컬러/폰트

기존 다크 테마 재사용. `src/components/ui/` 기존 Input/Button 컴포넌트 활용.
레이블은 `text-sm text-slate-400`, 입력은 `bg-slate-800 border-slate-700`.

## 빈 값 처리

UserProfile 레코드가 없으면 생성, 있으면 업데이트 (upsert).
모든 필드 optional. 저장 후 해당 값만 사용 지점에서 사용 (없으면 기본 공식 fallback).
