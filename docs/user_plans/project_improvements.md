## 🔴 긴급 (필수)

### 1. 최대심박수/LTHR 저장 및 활용
- **목표**: 사용자가 입력한 최대심박(176), LTHR(157) 저장 및 활용
- **작업**:
  - UserProfile에 `maxHR`, `lthr`, `lthrPace` 필드 추가
  - UI 편집 기능 추가
  - Activity 강도 분석 시 자동 Zone 계산 로직 추가
  - Zone 정의:
    ```
    Zone 1: <127 bpm (70%)
    Zone 2: 127-145 bpm (80%)
    Zone 3: 145-157 bpm (89%)
    Zone 4: 157-165 bpm (94%)
    Zone 5: 165-176 bpm (100%)
    ```
- **효과**: 모든 리포트의 강도 분석이 정확해짐

### 2. MyFitnessPal 식단 데이터 연동
- **목표**: 일일 섭취 칼로리를 자동으로 동기화
- **옵션 A (권장)**: MyFitnessPal API 직접 연동
  - 사용자 식단 데이터 자동 동기화
  - 단백질, 탄수화물, 지방 정보 수집
- **옵션 B**: Garmin Connect 경유 (Garmin API 확인 필요)
- **관련 모델**: FoodLog (이미 존재하지만 미사용)
- **효과**: 칼로리 밸런스 완전 분석 가능

### 3. 칼로리 밸런스 분석 기능
- **목표**: 섭취가능 칼로리(목표 + 활성) vs 실제 섭취 비교
- **계산 방식**:
  ```
  섭취가능 칼로리 = 목표 칼로리(1890) + 활성칼로리
  결손/잉여 = 섭취 - 섭취가능
  ```
- **작업**: DailySummary에 필드 추가
  - estimatedIntakeCalories (섭취)
  - availableCalories (섭취가능)
  - calorieBalance (차이)
- **효과**: 체중감량 진행도 명확화

## 🟡 높음 (권장)

### 4. Split/Lap 데이터 MCP 도구화
- **목표**: 구간별 러닝 분석을 쉽게 조회
- **내용**:
  - Lap별 거리, 시간, 페이스, 심박
  - 강도 타입 (WARMUP/INTERVAL/COOLDOWN)
  - 러닝 다이나믹스 (케이던스, 스트라이드, 지면접촉시간)
- **기존 API**: `/api/activities/[id]/splits` 존재
- **개선**: MCP 도구로 노출하여 조회 편의성 증대

### 5. 운동 강도 자동 분류
- **목표**: 사용자 해석 없이 자동으로 "Zone 4 운동", "한계치 런" 등 분류
- **작업**:
  - Activity에 필드 추가: zoneDistribution, estimatedZone, intensityScore
  - LTHR 기반 자동 분류 로직
  - 리포트에 자동 반영
- **효과**: 리포트 정확성 향상

### 6. 체중감량 진행 대시보드
- **목표**: 체중감량 진행 상황을 한눈에 보기
- **페이지**: `/dashboard/weight-loss`
- **내용**:
  - 체중 추세 (7일 이동평균)
  - 칼로리 밸런스 차트 (섭취 vs 소모)
  - 운동량 추세
  - 목표 vs 실제 진행도
  - 주간/월간 통계
